## Описание предметной области

**Шифр Плейфера** — ручной симметричный шифр, изобретённый Чарльзом Уитстоном в 1854 году и популяризированный бароном Плейфером. Это один из первых биграммных шифров, который шифрует текст парами букв, а не по одной.

### Принцип работы

1. **Построение матрицы 5×5:** из ключевого слова формируется матрица. Буквы ключа записываются без повторов, затем дописываются оставшиеся буквы латинского алфавита. Буквы I и J объединяются.

2. **Разбиение на биграммы:** текст разбивается на пары букв. Если в паре одинаковые буквы — между ними вставляется `X`. Если длина нечётная — в конец дописывается `X`.

3. **Правила шифрования каждой биграммы:**
    - **Одна строка** → каждая буква заменяется на следующую справа (циклически).
    - **Один столбец** → каждая буква заменяется на следующую снизу (циклически).
    - **Прямоугольник** → каждая буква заменяется на букву из того же ряда, но столбца второй буквы.

4. **Дешифрование** — обратные операции: влево вместо вправо, вверх вместо вниз.

### Пример

Ключ: `MONARCHY` → Матрица:

```
M  O  N  A  R
C  H  Y  B  D
E  F  G  I  K
L  P  Q  S  T
U  V  W  X  Z
```

Текст: `INSTRUMENTS` → Биграммы: `IN ST RU ME NT SX` → Шифротекст: `GATLMZCLRQTX`

## Средства отладки Visual Studio / Rider

В процессе разработки и отладки были использованы следующие встроенные средства:

| Средство | Применение |
|----------|-----------|
| **Breakpoints (точки останова)** | Установлены в методах `Encrypt`, `Decrypt`, `BuildMatrix` для проверки корректности промежуточных значений |
| **Step Over (F10)** | Пошаговое выполнение без входа в вызываемые методы |
| **Step Into (F11)** | Вход внутрь методов `PrepareBigrams` и `ProcessBigrams` для детальной проверки логики |
| **Watch (окно наблюдения)** | Отслеживание переменных: `bigrams`, `_matrix`, `_positions`, `sb.ToString()` |
| **Locals (локальные переменные)** | Просмотр всех переменных текущей области видимости на каждом шаге |
| **Immediate Window** | Ручная проверка выражений: `_positions['A']`, `_matrix[0,0]`, `Mod(-1, 5)` |
| **Call Stack (стек вызовов)** | Анализ цепочки вызовов при возникновении исключений |
| **Exception Settings** | Настройка перехвата `ArgumentException` и `ArgumentNullException` для проверки валидации |
